METHOD 2: BROWSER TESTING
--------------------------------------------------------------------------------

Step-by-step browser testing:

1. START THE API
   > python app.py
   Wait for: "Running on http://localhost:5000"

2. OPEN THE WEBSITE
   - Double-click index.html
   - Or drag it to your browser
   - Click "Prediction" in navigation

3. VISUAL CHECK
   ✓ Form is centered on page
   ✓ White card with shadow
   ✓ Title is large and clear
   ✓ All fields are visible
   ✓ Dropdowns show arrow icons
   ✓ Button is blue (#0071e3)

4. INTERACTION CHECK
   ✓ Click on temperature field → Blue glow appears
   ✓ Click on any dropdown → Opens smoothly
   ✓ Hover over button → Color changes
   ✓ Type in fields → Values update

5. FUNCTIONALITY TESTS

   Test A: Default Values
   - Leave all default values
   - Click "Predict Solar Irradiance"
   - Should show ~400 W/m² in large font
   - Should show interpretation text below
   - Result box should have gray background

   Test B: All Humidity Options
   - Keep other values same
   - Try each humidity option:
     * Very Low (20%)
     * Low (30%)
     * Moderate-Low (40%)
     * Moderate (50%)
     * Moderate-High (60%)
     * High (70%)
     * Very High (80%)
     * Extremely High (90%)
   - All should give predictions

   Test C: Different Times
   - Try these times:
     * 12:00 AM → Should give 0 W/m²
     * 6:00 AM → Should give low/zero
     * 12:00 PM → Should give high value
     * 1:00 PM → Should give high value
     * 6:00 PM → Should give low/zero
     * 11:00 PM → Should give 0 W/m²

   Test D: Different Months
   - Try summer months (Jun, Jul, Aug) → Higher values
   - Try winter months (Dec, Jan, Feb) → Lower values

   Test E: Error Handling
   - Stop Flask API (Ctrl+C in terminal)
   - Try to submit form
   - Should show red error box
   - Should say "Unable to connect"
   - Restart API and try again → Should work

6. CROSS-BROWSER TESTING
   Test in multiple browsers:
   - Google Chrome
   - Mozilla Firefox
   - Microsoft Edge
   
   In each browser:
   ✓ Form displays correctly
   ✓ Dropdowns work
   ✓ Predictions work
   ✓ No errors in console (Press F12)

7. MOBILE TESTING
   - Press F12 in browser
   - Click device toolbar icon (phone/tablet icon)
   - Try different sizes:
     * iPhone SE (375px)
     * iPad (768px)
   - Form should be readable and usable

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: "Cannot connect to API"
SOLUTION: 
1. Check Flask is running: python app.py
2. Check URL is http://localhost:5000
3. Check no firewall blocking port 5000

PROBLEM: Tests fail
SOLUTION:
1. Make sure Flask API is running
2. Check model files exist (random_forest_model.pkl, scaler.pkl)
3. Try: python save_model.py to regenerate models

PROBLEM: Wrong predictions
SOLUTION:
1. Verify model files are present
2. Check input values are in valid ranges
3. Re-run model training if needed

PROBLEM: Form doesn't work
SOLUTION:
1. Press F12 in browser
2. Check Console tab for errors
3. Verify script.js is loaded
4. Check API is running

PROBLEM: Dropdowns don't show options
SOLUTION:
1. Check browser compatibility
2. Try different browser
3. Clear browser cache
